## 文档用途
- 本文件是当前项目的前端主线总计划入口。
- 用于统一维护目标、优先级、阶段推进、分计划索引和风险依赖。
- 后续的分计划一律放在 `plan/` 目录下，不在根目录新增新的规划类文档。

## 当前现状
- 当前仓库已经落地后端基础代码，接口入口在 `backend/main.py`，接口前缀为 `/api/v1`。
- 当前仓库尚未落地 `frontend/` 目录，前端工程仍未初始化。
- **Agent 模块尚未开发**，`agent/` 目录不存在。这意味着：
  - 聊天助手回复始终是固定 Mock 文本（`"感谢您的输入。请继续补充更多信息。"`）
  - 案件要素提取/编辑功能无接口支撑（schema 有定义但路由未暴露）
  - 法律条文检索、案例搜索功能无接口支撑
  - 意图识别功能不可用
- 当前后端可用于前端联调的核心接口包括：
  - 创建会话
  - 获取会话
  - 发送消息（只返回用户消息，助手回复需再拉历史）
  - 获取消息历史
  - 生成文档（虽返回 202 但实际同步生成）
  - 获取文档
  - 健康检查
- 当前后端已知限制：
  - 聊天回复是 Mock 文本
  - 文档生成后 DB 中 `content` 字段始终为空（Mock 内容只写入文件系统）
  - 文档导出下载接口尚未真正实现，只返回占位 JSON

## 前端总目标
- 基于现有后端接口，完成前端 MVP 主链路。
- 前端不等待 Agent 或文档导出完全实现，先把用户可见流程跑通。
- 所有前端联调默认以实际代码和 `/docs` 为准，不以 README 规划描述替代真实协议。
- **前端 UI 需要达到正式产品水准**——使用现代设计风格，布局精致、交互顺畅，不能是简陋原型。

## MVP 主链路
1. 初始化前端工程。
2. 支持创建会话并进入聊天页面。
3. 支持展示消息历史、发送消息、展示助手回复。
4. 支持触发文档生成并展示生成状态。
5. 支持展示结果页，为后续下载功能预留入口和错误提示。

## 由于 Agent 未完成，前端当前做不了的事
- **案件要素结构化编辑**：后端无对应路由，schema 定义了但路由层未暴露。
- **法律条文检索 / 案例搜索**：后端无对应路由。
- **智能意图识别结果展示**：Agent 未接入，无此能力。
- **文档正文内容展示**：后端文档 `content` 字段始终为空，无法展示正文。
- **真实文件下载**：导出接口只返回 `{"message": "Download feature coming soon"}`。

前端设计需要对以上限制做好**优雅降级**——该有的 UI 入口和视觉元素要保留，但当前不可用的功能要给出诚实、友好的占位提示。

## 全局技术约定

### 页面路由表
```
/                    → 首页（案件类型选择 + 开始咨询）
/chat/:sessionId     → 聊天主页（消息列表 + 输入框 + 生成诉状入口）
/result/:docId       → 结果页（文档信息展示 + 下载占位）
```

### 布局策略
- 使用统一的 `MainLayout` 布局组件包裹所有页面。
- 布局结构：顶部导航栏 + 内容区。
- 顶部导航栏展示项目名称，可包含简单导航。
- 内容区由路由驱动，填充各页面内容。
- 整体视觉要现代、精致，参考主流法律服务类产品风格。

### API 服务层约定
- 统一使用 `fetch` 封装，不引入 axios 等重量级库。
- 在 `services/request.ts` 中创建基础请求封装：
  - 统一从环境变量读取 `VITE_API_BASE_URL`
  - 统一设置 `Content-Type: application/json`
  - 统一处理非 2xx 响应：抛出包含状态码和错误信息的自定义错误
  - 统一处理网络异常
- 各业务服务文件（如 `chatService.ts`、`documentService.ts`）调用基础封装。

### 错误提示约定
- 使用 Ant Design 的 `message.error()` 作为通用错误提示方式。
- 页面级错误（如会话不存在、文档不存在）使用 Ant Design `Result` 组件展示。
- 不使用 `notification`，保持风格统一。

## 风险与依赖
- 前端依赖当前后端接口结构，后端接口变动会直接影响前端实现。
- 聊天回复目前是 Mock，前端需要把展示和真实业务含义解耦。
- 文档导出接口未完成，前端当前只能先做状态展示和交互占位。
- 文档 `content` 字段始终为空（后端已知限制），前端不应预期能展示正文内容。
- Agent 模块未启动，案件要素、法律检索、意图识别等能力前端全部不可用。
- 当前 PowerShell / Conda 环境曾出现解释器路径问题，后续联调命令应继续确认实际使用的是项目环境。
