# 首页历史对话侧栏

## 目标
- 在首页左侧增加“历史对话”栏，展示用户已有会话。
- 支持点击历史会话直接进入对应聊天页。
- 为后续“继续上次咨询”“按最近活跃排序”“删除/重命名会话”等能力打基础。

## 当前状态
- 数据库已有 `sessions` 与 `messages` 表，可支撑历史会话功能。
- 当前前端首页仍是单列居中布局，尚无侧栏区域。
- 当前后端仅支持：
  - 创建会话
  - 获取单个会话
  - 获取单个会话消息历史
- 当前后端**缺少会话列表接口**，前端无法直接拿到首页历史会话数据。
- 2026-04-09 已完成第一版落地：首页左侧历史会话栏、会话列表接口与点击继续咨询链路均已打通。
- 2026-04-09 已完成第二版优化：历史栏独立滚动、删除会话能力、北京时间展示统一。

## 前置交付确认
- 阶段 1～4 已完成，现有首页、聊天页与会话创建主链路可用。
- 独立 Agent SSE 链路已恢复，可继续沿用现有聊天页逻辑。

## 范围
- 首页左侧历史会话列表 UI
- 后端会话列表接口
- 会话列表摘要字段设计（案件类型、最近更新时间、最近一条消息预览）
- 点击历史会话跳转到 `/chat/:sessionId`

## 任务拆解

### 1. 后端：补会话列表能力
- 在 `backend/api/schema.py` 新增首页历史会话响应模型，例如：
  - `SessionListItem`
  - `SessionListResponse`
- 在 `backend/services/chat.py` 新增查询方法：
  - 获取会话列表（按最近活跃倒序）
  - 获取每个会话的最近一条消息摘要
- 在 `backend/api/routes.py` 新增：
  - `GET /api/v1/sessions`

### 2. 后端：修正“最近活跃时间”
- 当前 `messages` 新增时，`sessions.updated_at` 不会自动刷新。
- 需要在 `ChatService.add_message()` 中同步更新所属 `Session.updated_at`。
- 否则首页历史列表无法按“最近聊天”正确排序。

### 3. 前端：补类型与服务层
- 在 `frontend/src/types/index.ts` 新增历史会话列表类型。
- 在 `frontend/src/services/chatService.ts` 新增：
  - `listSessions()`

### 4. 前端：首页布局改造
- 将首页从居中单列改成左右双栏：
  - 左侧：历史对话栏
  - 右侧：现有欢迎区 + 案件类型卡片 + 开始咨询按钮
- 推荐拆成独立组件，例如：
  - `frontend/src/components/SessionHistoryPanel.tsx`
  - `frontend/src/components/SessionHistoryPanel.css`

### 5. 前端：交互细节
- 空状态：暂无历史对话
- 加载态：骨架屏或轻量 Spin
- 点击条目：跳转对应聊天页
- 当前会话高亮：后续如需复用到聊天页再扩展

## 验收标准
- 首页左侧能展示历史会话列表
- 按最近活跃时间排序
- 每项至少展示案件类型、时间、最近消息摘要
- 点击会话可正确进入聊天页
- 无历史会话时有清晰空状态
- `npm run build` 通过

## 风险 / 阻塞
- 当前项目暂无用户体系，历史会话默认是“本地这个数据库里的全部会话”，后续若接入用户登录，需要补 `user_id` 过滤。
- 若直接为每个会话单独查一次最近消息，数据量大后会有性能问题；MVP 可先接受，后续可再优化 SQL。
- 首页双栏布局需要兼顾窄屏，移动端建议降级为顶部入口或折叠区。
